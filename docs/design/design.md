这份前端与接口设计文档为您定制了包含图谱构建、智能推理和双路检索问答三大核心模块的界面方案，以完全契合基于威胁情报的检索增强生成（RAG）系统的底层逻辑。

按照您的偏好，以下设计文档将不使用表格，并尽量精简额外说明，直接聚焦于功能、步骤、展示及API设计。

### 威胁情报检索增强生成系统 - 前端与接口设计文档

#### 界面一：威胁情报知识图谱构建界面(build)

**功能说明**
此界面主要用于支持非结构化安全分析报告的上传、智能分块、实体提取以及自动化构建底层知识图谱 。

* **入参 (Inputs)**
* **文件上传区**：支持多文件拖拽上传（格式限制：PDF, TXT, JSON 等）。
* **参数配置区**：文本分块长度阈值（默认4096 Tokens）、大模型基座选择（如 GPT-4o, DeepSeek-V3）。


* **交互步骤 (Steps)**
1. 用户上传原始威胁情报文档。
2. 系统对文件进行预处理（包括OCR识别）并按设定阈值进行文本分块 。


3. 调用大模型及提示词工程提取情报实体，并向 ATT&CK/CAPEC 框架进行拓展映射 。


4. 系统进行实体合并消歧，将生成的三元组持久化存储至图数据库。


* **结果展示 (Display)**
* **处理进度流**：实时滚动显示当前正在提取的文档块及抽取日志。
* **实体与关系列表**：以列表形式展示提取出的实体名称、类型及映射的 CAPEC_ID 等属性。
* **知识图谱可视化预览**：通过关系网络图动态渲染生成的节点（如恶意IP、漏洞CVE）与连边（如`exploit`, `mapped_to`）。



**后端 API 接口需求**

* **接口1：文件上传与预处理**
* **路径**：`/api/v1/kg/upload`
* **请求方式**：`POST`
* **请求体**：`multipart/form-data` (包含 file 文件流，`chunk_size` 整数)。
* **响应体**：返回处理成功的分块文档 ID 列表及基础元数据。


* **接口2：图谱抽取与构建**
* **路径**：`/api/v1/kg/extract`
* **请求方式**：`POST`
* **请求体**：包含 `document_ids` (列表), `llm_model` (字符串)。
* **响应体**：返回提取的 `entities` (包含实体属性) 和 `triples` (包含头尾节点及关系)，以及图谱构建状态。

#### 界面二：知识图谱智能推理补全界面(inference)

**功能说明**
此界面用于加载局部知识子图，并利用训练好的因果强化学习代理（RL Agent）执行多跳推理，以发现并补全隐藏的攻击链路与关系边 。

* **入参 (Inputs)**
* **目标实体检索框**：输入核心实体名称或 ID（如特定 APT 组织或受害资产）。
* **推理参数调节**：最大推理跳数限制（滑动条，默认6跳）、因果偏差阈值。


* **交互步骤 (Steps)**
1. 用户输入目标实体并点击“加载子图”。
2. 系统通过广度优先搜索从全局图谱中截取局部推理环境 。


3. 用户点击“执行因果推理”。
4. RL代理在子图环境中进行前向计算，预测出极有可能存在的缺失关系边 。




* **结果展示 (Display)**
* **前后对比图谱渲染**：左侧展示原始局部知识子图，右侧展示推理后的子图。
* **高亮增强标记**：在右侧图谱中，使用特殊颜色（如红色虚线）及动画效果高亮标出代理新推断出的关系链路（如“推断：初始访问 -> 执行”）。
* **推理日志面板**：输出代理每一步推演的反事实干预效应得分及动作概率。



**后端 API 接口需求**

* **接口1：局部子图加载**
* **路径**：`/api/v1/reasoning/subgraph`
* **请求方式**：`GET`
* **请求参数**：`entity_id` (字符串), `max_hops` (整数)。
* **响应体**：返回以该实体为中心的局部子图的节点与边数据格式（供前端图表库渲染）。


* **接口2：强化学习前向推理**
* **路径**：`/api/v1/reasoning/predict`
* **请求方式**：`POST`
* **请求体**：包含当前子图的结构数据 `subgraph_data`。
* **响应体**：返回预测出的候选关系边列表 `predicted_edges` 及其因果置信度得分。

#### 界面三：双路协同智能检索问答界面

**功能说明**
此界面是最终面向安全分析人员的对话终端，融合了向量相似度检索与图谱拓扑检索，提供高准确率和强可解释性的威胁情报问答服务 。

* **入参 (Inputs)**
* **对话输入框**：支持自然语言输入查询（例如：“分析某漏洞被何种工具利用并导致了哪些资产受损？”）。
* 
**权重调节器**：检索分数权重因子 $\alpha$ 的滑块调节（0.0 ~ 1.0），用于控制图谱实体得分与向量相似度得分的混合比例 。




* **交互步骤 (Steps)**
1. 用户输入安全问题并发送。
2. 系统后台并行触发：问答语句的文本嵌入向量化检索，以及问答语句实体提取后的图谱多跳关联检索 。


3. 系统对两路召回的文档块进行加权混合重排 。


4. 大语言模型基于重排后的优质上下文生成安全研判回复 。




* **结果展示 (Display)**
* **对话流展示**：类似于主流大模型的聊天界面，流式输出回答文本。
* **溯源引用面板 (Citations)**：在回答的下方或侧边栏，折叠展示当前回答所引用的“原始情报文档块摘要”以及“图谱关联推理路径”，点击可查看依据详情。



**后端 API 接口需求**

* **接口1：混合检索与增强生成**
* **路径**：`/api/v1/rag/query`
* **请求方式**：`POST`
* **请求体**：包含 `user_query` (字符串), `alpha_weight` (浮点数), `top_k` (整数)。
* **响应体**：
* `answer`: 大模型生成的最终自然语言文本。
* `citations`: 包含 `vector_docs` (向量召回的文档块明细) 和 `graph_paths` (图谱召回的关联实体链路明细)。
